## 1. 목적

"몬스터 길들이기 스타 다이브"의 일일/주간 숙제를 모바일 브라우저에서 체크하기 위한 **1인용 정적 웹앱**. 서버·로그인·동기화 없음.

## 2. 배포 형태

- **단일 파일** `index.html` — HTML + CSS + JS + 이모지 전부 인라인.
- `file://` 로도, 정적 호스팅으로도 둘 다 동작.
- 더블클릭하면 즉시 실행.

## 3. 저장소

- `localStorage` 키 하나: `mongil_state`
- 스키마:

```json
{
  "baselineDate": "2026-04-23",
  "weeklyBaselineWeek": "2026-W17",
  "cheatsheetOpen": false,
  "checked": {
    "login": true,
    "quest": false,
    "capture": false,
    "reward": false,
    "dungeon": false,
    "fieldboss": false,
    "rift": false
  }
}
```

- `checked` 의 키는 안정적인 영문 ID(내부용). 화면에는 한글 라벨이 별도로 매핑됨.
- `baselineDate`: 일일 리셋 기준 (KST 09:00).
- `weeklyBaselineWeek`: 주간 리셋 기준 (KST 월요일 09:00 기준 ISO week).
- `cheatsheetOpen`: 치트시트 섹션 접힘/펼침 상태.
- 손상/결측 시 기본값 `{ baselineDate: <계산값>, weeklyBaselineWeek: <계산값>, cheatsheetOpen: false, checked: {} }` 로 초기화.

## 4. 기준일 계산 (리셋 로직)

게임 리셋은 **매일 09:00 KST**.

```
KST 현재 시각 = UTC 시각 + 9시간 (수동 오프셋, 브라우저 로컬타임 신뢰 X)
  kst.hour < 9  → baseline = (KST 기준) 어제 YYYY-MM-DD
  kst.hour ≥ 9  → baseline = (KST 기준) 오늘 YYYY-MM-DD
```

- 페이지 로드 시 `storage.baselineDate !== computedBaseline` 이면 `checked` 를 `{}` 로 리셋하고 `baselineDate` 갱신.
- 페이지를 오래 열어둔 채 09:00이 지나는 경우도 체크 액션마다 또는 1분 주기로 기준일 재평가.

### 주간 항목 리셋
- 주간 키(`rift` 등)는 일일과 **같은 `checked` 맵**에 넣되, 리셋 조건만 분리.
- 판정: 저장된 `weeklyBaselineWeek` ≠ 현재 KST 기준 계산된 주 번호 → 주간 키만 `false` 로.
- 주 번호: KST 월요일 09:00 기준 ISO week (예: `"2026-W17"`).
- 일일 리셋과 주간 리셋은 독립적으로 판정 — 일요일→월요일 09:00에는 둘 다 발동, 평일엔 일일만.

## 5. 체크 항목 (확정 리스트)

### 일일 (6개)
| key | 라벨 | 부가설명 |
|---|---|---|
| `login` | 로그인 | 공헌도 +20 |
| `quest` | 의뢰 5개 완수 | 공헌도 +60, 별빛 25×5 |
| `capture` | 필드 몬스터 5마리 포획 | 공헌도 +40 |
| `reward` | 일일 공헌도 100점 보상 수령 | 별빛 50, 골드 5만 (7일 이월 후 소멸) |
| `dungeon` | 던전 10판 | 열쇠 소모, 최고난도 1클 보너스 |
| `fieldboss` | 필드 보스 처치 | 도감·네임드 몬스터링 |

### 주간 (1개)
| key | 라벨 | 부가설명 |
|---|---|---|
| `rift` | 균열 5판 + 입장권 구매 | 월요일 09:00 리셋 |

### 인터랙션
- **카드 전체가 터치 영역** (체크박스만 누르기 어려운 손가락 UX 회피).
- 체크 상태 토글 시: 즉시 localStorage 반영 + 진행률 갱신.
- 치트시트 섹션 헤더 탭 → 접힘/펼침 토글. 상태는 localStorage에 함께 저장(`cheatsheetOpen: true/false`).

### 진행률
- 상단에 `n/m 완료` + 얇은 프로그레스 바. 일일 + 주간 **총합** 기준.

## 8. 엣지케이스

- **localStorage 비거나 JSON 파싱 실패** → 기본값으로 초기화.
- **탭 여러 개 열었을 때** → `storage` 이벤트 리스너로 체크 상태 동기화.
- **페이지를 오래 열어둠 → 리셋 시각 통과** → 매 체크 액션마다, 그리고 1분 주기 setInterval로 기준일 재평가.
- **브라우저 시스템 시각 오차** → 신뢰 (본인 1인 앱).
- **화면 회전 / 가로 모드** → 카드 세로 스택 유지, 가로 폭만 확장.

## 9. 스코프에서 제외 (YAGNI)

- 로그인 / 서버 / 동기화 — 본인 1인 기기.
- 항목 추가/삭제 UI — JS 배열 직접 수정으로 충분. 게임 버전업은 사용자 소관.
- PWA / 푸시 알림 — `file://` 에서 Service Worker 등록 불가, 1인 앱이라 과함.
- 이모지 외 커스텀 아이콘 번들 — 이미지 교체는 사용자가 base64로 수동 교체.
